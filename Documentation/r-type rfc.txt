2.2.  Client Response

   Client MAY send a response after receiving a packet.  The response
   SHOULD be composed of the magic number and a response OK / KO.
   Optionally we can add a message to details the answer.

2.3.  Server Response

   Server MUST send a response after receiving a packet.  The response
   SHOULD be composed of the magic number and a response OK / KO.
   Optionally we can add a message to details the answer.

3.  Server to Client Command

3.1.  Alive

   Check if client is still connected.

3.2.  LaunchGame

   Start game for room and wait for entity data.

3.3.  Update

   Update all data of game.

4.  Client to Server Command

4.1.  Join Room

   Ask to join a room.

4.2.  Leave Room

   Leave a room.

4.3.  Ready

   Set client status to ready then start game when all player in room
   are ready.
